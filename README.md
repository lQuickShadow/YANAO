# Информационная система службы технической поддержки

## Наименование проекта

Полное наименование проекта: **Информационная система службы технической поддержки отдела информатизации ГБПОУ ЯНАО «Ямальский многопрофильный колледж»**.

Система реализована в виде настольного приложения на Python с использованием Tkinter и ORM Peewee.

## План реализации

1. Разработка диаграмм вариантов использования и классов.
2. Проектирование и создание базы данных MySQL.
3. Разработка структуры проекта (модели, контроллеры, представления).
4. Реализация функционала по ТЗ:
   - аутентификация и роли (RBAC),
   - управление заявками (Helpdesk),
   - база знаний,
   - управление пользователями,
   - базовая статистика/дашборд.

## Структура базы данных

Используется СУБД **MySQL**, доступ к БД — через `Connect/connect.py`, модели описаны с помощью Peewee.

### Users (пользователи)

- `id` – первичный ключ (`PrimaryKeyField`).
- `name` – отображаемое имя пользователя.
- `role` – внешняя ссылка на таблицу `Role` (роль пользователя).
- `login` – логин пользователя, **уникальный**.
- `password` – хешированный пароль (bcrypt).

### Role (роли)

- `id` – первичный ключ.
- `name` – наименование роли.
- Предусмотрены константы:
  - `USER` – Пользователь,
  - `SPECIALIST` – Специалист,
  - `ADMIN` – Администратор.

### Ticket (заявки)

- `id` – первичный ключ.
- `title` – заголовок заявки (включает пометку срочности).
- `description` – подробное описание проблемы (включает строку со срочностью).
- `status_id` – внешний ключ на `Status` (статус заявки).
- `created_at` – дата и время создания заявки.
- `updated_at` – дата и время последнего обновления.
- `user_id` – внешний ключ на `Users` (автор заявки).
- `executor_id` – внешний ключ на `Users` (исполнитель заявки).
- `category_id` – внешний ключ на `Category` (категория проблемы).

### Category (категории заявок)

- `id` – первичный ключ.
- `title` – название категории (уникальное).
- `description` – описание категории (опционально).
- Фиксированные категории (инциденты, ПО, доступ, консультации и др.) создаются в `create_table.py`.

### Status (статусы заявки)

- `id` – первичный ключ.
- `name` – название статуса.
- Фиксированные значения:
  - `NEW` – «Новый»,
  - `IN_PROGRESS` – «В работе»,
  - `DONE` – «Завершено».

### Comment (комментарии)

- `id` – первичный ключ.
- `description` – текст комментария.
- `user_id` – внешний ключ на `Users` (кто оставил комментарий).
- `ticket_id` – внешний ключ на `Ticket` (к какой заявке относится).
- `type_id` – внешний ключ на `Type` (тип комментария: публичный / внутренний).

### ArticleBase (база знаний)

- `id` – первичный ключ.
- `title` – заголовок статьи.
- `description` – текст статьи (решение/инструкция).
- `ticket_id` – внешний ключ на `Ticket` (заявка, из которой родилась статья).
- `executor_id` – внешний ключ на `Users` (специалист – автор статьи).

### Type (типы комментариев)

- `id` – первичный ключ.
- `name` – название типа.
- Константы:
  - `PUBLIC` – публичный комментарий (виден пользователю),
  - `INTERNAL` – внутренний комментарий (для тех. специалистов и администратора).

## Используемые технологии

- **Python** – язык реализации приложения.
- **Tkinter** – библиотека для создания настольного графического интерфейса.
- **MySQL** – СУБД для хранения данных.
- **Peewee** – ORM для работы с MySQL.
- **mysqlclient / mysql-connector-python** – драйверы подключения к БД.
- **bcrypt** – хеширование паролей пользователей.

## Установка и запуск
Для начала нужно создать БД для этой ИС 

1. Установить зависимости из `packages.txt` (и `bcrypt`, если ещё не установлен):

   ```bash
   pip install -r packages.txt
   pip install bcrypt
   ```

2. Настроить подключение к MySQL в файле `Connect/connect.py` (имя БД, логин, пароль, хост, порт).

3. Создать таблицы и заполнить справочники (роли, статусы, категории, типы комментариев):

   ```bash
   python Models/create_table.py
   ```

4. Запустить приложение:

   ```bash
   python main.py
   ```

   При запуске откроется окно авторизации. Из него можно перейти в регистрацию нового пользователя.

## Основной функционал

### Пользователи и роли

Контроллер: `UserController` (`Controllers/user_controller.py`).

- Регистрация пользователей с хешированием пароля (bcrypt).
- Авторизация по логину и паролю.
- Роли:
  - **Пользователь** – создаёт заявки и просматривает свои, доступ к базе знаний.
  - **Специалист** – видит и обрабатывает назначенные ему заявки, доступ к базе знаний.
  - **Администратор** – управляет всеми заявками, пользователями и просматривает статистику.

### Работа с заявками

Контроллер: `TicketController` (`Controllers/ticket_controller.py`), окна `main_view.py`, `create_ticket_view.py`, `ticket_detail_view.py`, `assign_ticket_view.py`.

- Создание заявок пользователем и администратором:
  - тема,
  - описание,
  - категория,
  - пометка срочности (отображается в заголовке/описании).
- Просмотр списка заявок:
  - пользователь – только свои заявки,
  - специалист – только назначенные ему,
  - администратор – все заявки.
- Назначение заявки специалисту (окно назначения, доступно админу).
- Изменение статуса:
  - при назначении – перевод в статус «В работе»,
  - завершение заявки – перевод в статус «Завершено».

### Комментарии

Контроллер: `CommentController` (`Controllers/comment_controller.py`), окно `ticket_detail_view.py`.

- Добавление комментариев к заявке.
- Публичные комментарии видны пользователю, внутренние – только специалистам и администратору.

### База знаний

Контроллер: `ArticleController` (`Controllers/article_controller.py`), окна `article_create_view.py`, `kb_view.py`.

- Создание статьи на основе решённой заявки.
- Просмотр и поиск статей по названию.
- Просмотр базы знаний пользователями и специалистами.

### Управление пользователями

Окно: `users_view.py` (для администратора).

- Просмотр списка пользователей.
- Удаление пользователей.

### Статистика

Контроллер: `StatisticsController` (`Controllers/statistics_controller.py`), окно `stats_view.py`.

- Показ:
  - общего количества заявок,
  - количества завершённых заявок,
  - количества заявок в работе.
